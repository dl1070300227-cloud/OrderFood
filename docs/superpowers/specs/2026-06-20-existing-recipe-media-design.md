# 已有菜品图文视频教程设计

## 目标

把已经入库的常见菜教程从单段文字升级为更容易阅读的结构化步骤，并为每道菜提供真实视频平台入口。升级过程不得覆盖用户已经手动编辑过的图文步骤、图片或视频链接。

## 范围

- 已有 128 道常见菜在服务启动时自动补齐富媒体教程数据。
- 新增入库的常见菜也应直接带上结构化步骤和视频入口。
- 视频只保存搜索入口链接，不下载、不转存、不解析第三方视频。
- 图片字段继续保留给用户手动上传；本次不批量抓取第三方图片，避免版权和热链稳定性问题。

## 数据规则

- `recipes.video_url` 为空时，生成 B 站搜索入口：`https://search.bilibili.com/all?keyword=<菜名> 做法`。
- `recipe_steps` 为空时，将 `recipes.steps` 拆成多个步骤。
- 如果某菜已有多条步骤、步骤图片、封面图或视频链接，则视为用户已维护，不覆盖。
- 如果只有一条系统迁移出来的旧步骤，且内容等于 `recipes.steps`，允许替换为拆分后的多步骤。

## 展示效果

点开“教程”后，已有菜品至少显示：

- 食材和调料。
- “打开视频教程”按钮。
- 多个步骤卡片，每张卡片包含步骤序号和说明。

图片仍可在“菜品管理”中手动上传到封面图或步骤图。

## 测试

- 后端测试验证种子菜会返回视频入口和多条结构化步骤。
- 后端测试验证用户已维护的视频/步骤不会被二次回填覆盖。
- 前端既有教程展示测试继续覆盖视频入口和图文步骤渲染。
