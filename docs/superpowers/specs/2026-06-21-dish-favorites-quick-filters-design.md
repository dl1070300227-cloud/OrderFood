# 菜品喜爱与快捷筛选设计

## 背景

当前首页顶部的“分类”和“推荐”卡片只展示数量，视觉上像可点击卡片，但不能操作。点菜工作台中已有搜索和分类下拉，但用户需要更直接的入口：点击顶部栏目即可查看对应菜品。同时，每道菜需要一个“爱心”标记，方便家庭成员把常吃、喜欢的菜收进爱好列表。

## 目标

- 顶部概览卡片从纯展示变成快捷筛选入口。
- 每道菜支持添加或取消爱心喜爱状态。
- 新增“爱好”栏目，点击后只查看已喜爱的菜品。
- 喜爱状态保存到数据库，电脑和手机局域网访问看到一致结果。

## 功能范围

### 快捷筛选

顶部概览区包含：

- 菜单：查看全部菜品。
- 分类：进入点菜页并聚焦分类筛选，默认仍显示全部分类。
- 推荐：只查看 `isRecommended = true` 的菜品。
- 爱好：只查看 `isFavorite = true` 的菜品。
- 待做：保留跳转订单记录页。

快捷筛选和点菜工作台内的搜索、分类下拉可以叠加使用。

### 爱心喜爱

- 每张菜品卡片右上区域增加爱心按钮。
- 未喜爱使用空心爱心，已喜爱使用实心视觉状态。
- 点击后立即请求后端保存，成功后刷新菜品列表。
- 请求失败时显示当前工作台消息提示。

## 后端设计

### 数据库

`dishes` 表新增：

- `is_favorite INTEGER NOT NULL DEFAULT 0`

启动时执行兼容迁移，旧数据库自动补列。

### API

`PATCH /api/dishes/:id/favorite`

请求：

```json
{
  "isFavorite": true
}
```

响应：更新后的菜品对象。

错误：

- 菜品不存在返回 `404`。
- 请求体非法返回 `400`。

## 前端设计

- `Dish` 和 `DishInput` 增加 `isFavorite`。
- `DishWorkspace` 增加 `quickFilter` 参数。
- `App` 管理当前快捷筛选状态，并把顶部概览卡片改成按钮。
- 菜品卡片增加爱心按钮，按钮具有 `aria-pressed` 和明确的可访问名称。
- “推荐”和“爱好”筛选选中时，顶部按钮和工作台标题显示当前筛选状态。

## UI/UX 约束

- 使用 Lucide `Heart`、`ClipboardList`、`Star` 等现有图标体系。
- 顶部卡片使用原 8px 圆角和原色板，只增加按钮语义、悬停态、选中态。
- 所有可点击元素最小高度不低于 44px。
- 移动端顶部卡片保持单列，不产生横向滚动。
- 不使用 emoji 作为结构图标。

## 测试策略

后端：

- 列表返回 `isFavorite`。
- `PATCH /api/dishes/:id/favorite` 可设置和取消喜爱。
- 菜品不存在返回 `404`。

前端：

- 点击顶部“推荐”后只展示推荐菜。
- 点击顶部“爱好”后只展示喜爱菜。
- 点击菜品爱心按钮调用后端接口并刷新菜品。

## 非目标

- 不增加多用户喜好。
- 不做复杂标签系统。
- 不做按分类卡片展开分类列表的二级导航。
