# 订单删除与消费统计设计

## 背景

当前家用点菜小程序已经支持创建订单、查看订单记录、切换订单完成状态。用户希望订单记录页继续补齐两个高频家庭使用能力：

- 删除误点、重复点或不需要保留的订单记录。
- 查看消费统计，包括今日消费、本月消费、年度消费和自定义时间段消费。

## 目标

在订单记录页内完成订单管理和消费概览，不新增独立报表页。后端提供统一的统计口径，前端只负责展示和传入时间范围。

## 功能范围

### 订单删除

- 每条订单记录提供删除入口。
- 删除前弹出浏览器确认，避免误删。
- 删除订单时同步删除该订单的 `order_items` 明细。
- 删除成功后刷新订单列表和消费统计。
- 删除不存在的订单返回 `404`。

### 消费统计

订单记录页顶部增加消费统计区域，包含：

- 今日消费。
- 本月消费。
- 年度消费。
- 自定义时间段消费。

自定义时间段使用两个日期输入框：

- 开始日期：`YYYY-MM-DD`。
- 结束日期：`YYYY-MM-DD`。
- 如果开始日期晚于结束日期，前端提示用户修正，不发起统计请求。

## 统计口径

- 统计字段：`orders.total_price`。
- 时间字段：`orders.ordered_at`。
- 统计对象：所有未删除订单，不区分 `pending` 和 `completed` 状态。
- 日期范围：按本地自然日计算，开始日期包含当天 `00:00:00.000`，结束日期包含当天 `23:59:59.999`。
- 删除订单后，该订单不再出现在列表和任意统计结果中。

## 后端接口

### 删除订单

`DELETE /api/orders/:id`

- 成功：`204 No Content`。
- 不存在：`404 { message: "订单不存在" }`。

### 查询消费统计

`GET /api/orders/stats?startDate=YYYY-MM-DD&endDate=YYYY-MM-DD`

响应：

```json
{
  "total": 128.5,
  "orderCount": 3,
  "startDate": "2026-06-01",
  "endDate": "2026-06-21"
}
```

校验规则：

- `startDate` 和 `endDate` 必须同时传入。
- 日期必须是 `YYYY-MM-DD` 格式。
- `startDate <= endDate`。
- 参数不合法返回 `400`。

前端会分别请求今日、本月、年度和自定义时间段统计。

## 前端交互

- `OrderHistory` 接收订单列表、统计数据、删除回调、状态更新回调。
- 每条订单保留现有“标记完成/待完成”按钮，并新增删除按钮。
- 消费统计显示为紧凑卡片，移动端单列展示，桌面端四列展示。
- 自定义统计区域放在统计卡片下方，日期输入和查询按钮在移动端自动换行。
- 删除和统计请求失败时沿用当前全局消息提示。

## 数据流

1. 应用启动时同时加载菜品、订单和默认统计。
2. 用户进入订单记录页后能看到最近订单和统计卡片。
3. 用户删除订单：
   - 前端确认。
   - 调用 `DELETE /api/orders/:id`。
   - 成功后重新加载订单和统计。
4. 用户选择自定义日期：
   - 前端校验日期顺序。
   - 调用统计接口。
   - 更新自定义消费卡片。

## 测试策略

后端：

- 删除订单成功后列表不再返回该订单。
- 删除不存在订单返回 `404`。
- 统计接口能按日期范围汇总金额和订单数。
- 非法日期范围返回 `400`。

前端：

- 订单记录页点击删除会调用删除接口并刷新。
- 统计卡片显示今日、本月、年度和自定义消费。
- 自定义开始日期晚于结束日期时显示提示，不调用统计接口。

## 非目标

- 不做软删除或回收站。
- 不做按菜品、分类、家庭成员的统计。
- 不新增图表库。
- 不把 `pending` 订单排除在统计外。
